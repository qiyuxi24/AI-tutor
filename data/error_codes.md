# AI Tutor 错误码速查表

> 当网页对话框中出现 `[E-XXX-NNN]` 格式的错误码时，可在此表中查找原因和排查方向。

---

## E-LLM（大模型 API 调用层）

> SDK → 阿里云千问的调用过程。错误来自 OpenAI SDK 抛出的异常类型。

| 错误码 | 用户提示 | 可能原因 | 排查方向 |
|--------|---------|---------|---------|
| E-LLM-001 | AI服务响应超时 | 千问 API 在 120s 内未返回结果 | 检查千问服务状态、网络延迟 |
| E-LLM-002 | AI服务请求过于频繁 | 触发千问 API 限流（QPM/TPM） | 降低请求频率，检查配额 |
| E-LLM-003 | AI服务认证失败 | DASHSCOPE_API_KEY 无效或过期 | 检查 `.env` 中的 API Key |
| E-LLM-004 | 无法连接到AI服务 | 网络不通或 DNS 解析失败 | 检查能否 ping `dashscope.aliyuncs.com` |
| E-LLM-005 | AI服务内部错误 | 千问服务端 5xx 错误 | 稍后重试，检查千问状态页 |
| E-LLM-006 | AI返回了空回复 | LLM 返回 content 和 tool_calls 均为空 | 检查 prompt 是否合理，重试 |
| E-LLM-007 | 知识图谱工具执行失败 | function calling 工具执行出错 | 检查 `llm_client.py` 中 `execute_kg_tool` 日志 |

---

## E-COMM（前后端通信层）

> 前端 axios ↔ 后端 FastAPI 之间的通信错误。这些错误码由前端 catch 块生成，用于区分"AI 调用失败"和"通信链路故障"。

| 错误码 | 用户提示 | 可能原因 | 排查方向 |
|--------|---------|---------|---------|
| E-COMM-001 | 请求超时，后端处理过慢或网络延迟 | axios 超时（前端等待后端响应超过限制） | 检查 LLM 调用耗时、网络延迟、Vite 代理超时配置 |
| E-COMM-002 | 无法连接到后端服务器 | 后端未启动或端口不通 | 确认 FastAPI 已启动在 localhost:8000 |
| E-COMM-003 | 后端网关错误 (502) | Nginx/Caddy 等反向代理无法连接后端 | 检查反向代理配置和后端进程状态 |
| E-COMM-004 | 后端服务不可用 (503) | 后端过载或正在重启 | 等待片刻后重试 |
| E-COMM-005 | 后端网关超时 (504) | 反向代理等待后端超时 | AI 调用可能仍在处理中，可适当增大代理超时 |
| E-COMM-006 | 后端内部错误 (5xx) | FastAPI 未捕获的异常 | 查看后端控制台日志 |
| E-COMM-007 | 后端返回了未预期的响应 | HTTP 状态码异常（非 5xx 也非 2xx） | 检查请求参数和路由配置 |

## E-GRAPH（图谱分析器）

| 错误码 | 用户提示 | 可能原因 | 排查方向 |
|--------|---------|---------|---------|
| E-GRAPH-001 | 对话分析失败 | 分析 LLM 调用异常（超时/网络等） | 查看日志中具体异常信息 |
| E-GRAPH-002 | 分析结果格式异常 | LLM 返回了非 JSON 内容 | 查看日志中原始响应前 200 字 |
| E-GRAPH-003 | 未检测到有效的图谱更新建议 | 分析结果中无有效建议或全部为 none | 正常情况，无需处理 |

---

## E-KG（知识图谱核心）

| 错误码 | 用户提示 | 可能原因 | 排查方向 |
|--------|---------|---------|---------|
| E-KG-001 | 知识图谱文件加载失败 | `data/knowledge/index.json` 损坏或不存在 | 检查 JSON 格式，尝试手动修复 |
| E-KG-002 | 知识图谱保存失败 | 磁盘空间不足或权限问题 | 检查 `data/knowledge/` 目录权限 |
| E-KG-003 | 目标知识点不存在 | 操作了不存在的节点 ID | 确认节点 ID 拼写正确 |
| E-KG-004 | 知识点已存在 | 尝试创建重复节点 | 检查是否已有同名节点 |
| E-KG-005 | 目标关系边不存在 | 操作了不存在的边索引 | 刷新图谱后重试 |
| E-KG-006 | 关系边已存在 | 相同的 from+to+relation 边已存在 | 检查图谱中现有关系 |
| E-KG-007 | 无效的图谱操作 | 操作参数不合法 | 查看日志中具体报错 |

---

## E-CHAT（对话服务层）

| 错误码 | 用户提示 | 可能原因 | 排查方向 |
|--------|---------|---------|---------|
| E-CHAT-001 | 提示词构建失败 | 模板文件缺失或渲染异常 | 检查 `data/prompts/` 目录下的模板文件 |
| E-CHAT-002 | 对话处理异常 | 对话流程编排中的意外错误 | 查看日志中的完整堆栈 |
| E-CHAT-003 | 对话后分析失败 | 图谱分析阶段出错（不影响对话） | 查看日志中 `graph_analyzer` 相关错误 |

---

## E-EVENT（事件总线）

| 错误码 | 用户提示 | 可能原因 | 排查方向 |
|--------|---------|---------|---------|
| E-EVENT-001 | 事件发布失败 | 事件队列异常 | 检查 SSE 连接是否正常 |
| E-EVENT-002 | 事件队列已满 | 事件消费速度跟不上生产速度 | 检查前端 SSE 连接 |
| E-EVENT-003 | SSE连接异常 | 前端 SSE 断连或解析错误 | 检查浏览器控制台 Network 面板 |

---

## E-SYS（通用/系统层）

> 纯系统级错误，不归属任何业务模块。

| 错误码 | 用户提示 | 可能原因 | 排查方向 |
|--------|---------|---------|---------|
| E-SYS-001 | 文件读写异常 | 磁盘 I/O 错误 | 检查磁盘空间和文件权限 |
| E-SYS-002 | 未知系统错误 | 未分类的异常 | 查看日志中的完整堆栈信息 |

---

## 日志文件位置

- **后端日志**：FastAPI/Uvicorn 控制台输出（终端）
- **日志格式**：`2026-05-29 23:59:59 | ERROR   | ai-tutor | E-LLM-001 | AI服务响应超时 | detail=...`
- **前端日志**：浏览器开发者工具 Console（SSE 错误事件会输出 `[SSE Error]` 前缀）

## 调试技巧

1. **看到 `[E-LLM-xxx]`** → AI 调用链路问题，优先检查千问 API 可用性和 Key
2. **看到 `[E-COMM-xxx]`** → 前后端通信问题，检查后端是否启动、网络是否通、超时配置是否合理
3. **看到 `[E-GRAPH-xxx]`** → 不影响对话，检查分析器日志
4. **看到 `[E-KG-xxx]`** → 检查 `data/knowledge/index.json` 完整性
5. **看到 `[E-SYS-xxx]`** → 检查后端控制台完整堆栈
