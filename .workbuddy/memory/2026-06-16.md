## 用户数据隔离三处修复

### 审查发现
JWT→API→DB 鉴权链路通畅（14个CRUD端点全部正确鉴权，SQL WHERE user_id=? 隔离完整），
但存在3个数据隔离漏洞。

### Fix1: 前端 localStorage 按 user_id 隔离
- **文件**: frontend/src/stores/chatStore.js
- **问题**: localStorage key 固定为 `ai_tutor_conversations`，切换账号后对话历史泄露
- **修复**: key 改为 `ai_tutor_conversations_{userId}` 格式，通过 localStorage 中 `ai_tutor_user` 读取 user_id
- **连动修复**: connectSSE() 增加 token 检查，未登录时不连接

### Fix2: MD 知识文件按 user_id 隔离
- **文件**: backend/app/core/knowledge_graph.py + backend/app/api/v1/knowledge.py
- **问题**: 所有用户的 MD 节点文件存在同一 `data/knowledge/` 目录，同 ID 节点互相覆盖
- **修复**: KnowledgeGraph 新增 `nodes_dir` 属性 → `data/knowledge/nodes/{user_id}/`，自动创建目录
- knowledge_graph.py 3处引用 + knowledge.py 4处引用全部更新

### Fix3: SSE events 端点强制认证
- **文件**: backend/app/api/v1/knowledge.py
- **问题**: `/knowledge/events` 的 token 参数为可选，未认证也可订阅全局广播
- **修复**: 签名改为 `user_id: int = Depends(get_current_user_from_token)`，无 token 或 token 无效返回 401
